Group Browser

To Build: run "nmake" or launch the .sln file in Visual Studio .Net

Using the Group Browser Sample Application:

The Group Browser Sample Application demonstrates how to:

1. Create a new Identity
To create a new identity, simply right click on the 'Identities' tree node and select 'Create New Identity'. Next, enter a 'Friendly Name' and 'Classifier' in the pop up dialog box and click OK. The new identity will appear in the tree view. Selecting the identity will display its properties in the right hand panel.

2. Delete an Identity
To delete an identity, right click on the identity to delete and select 'Delete Identity'. A pop up dialog box will appear confirming the deletion. Click ‘Yes’. The identity is then removed from the system and the tree view. NOTE: You may not delete an identity that has a group associated with it. To properly delete an identity, you must first delete the group it is associated with, then delete the identity.

3. Create a Group
In order to create a group, right click on an identity and select 'Create Group' from the submenu. Enter 'Group Friendly Name' and choose a Cloud in which to create the group (A drop down list will list available the names of available clouds in Link-local, Site-local, and Global scopes).  You can also enter a 'Group Classifier', 'Group Comment' if desired.  Click 'OK'. The new group will appear under the identity selected.

4. Delete a Group
To delete a group, simply right click on the group to delete and select 'Delete Group' from the submenu. A pop up dialog box will appear confirming the deletion. Click the 'Yes' button. The group selected will be removed from the system and the tree view.

5. Create Identity Information
The first step is to create the identity information necessary for issuing an invitation.  In the tree view, select the identity being invited into a group. Right click on the identity and select ‘Save Identity Information'. A pop up dialog will appear asking where to save the information. Click on the 'Browse' button and select a location. Enter a filename to use in the 'Filename:' edit box and click 'OK'. Click 'OK' again at the ‘Save Identity Information’ dialog. The identity information must now be transferred to the machine of an individual already in the group.

6. Issue an Invitation
Once the identity information file has been transferred via an email or file share to the machine of a peer in the group, an invitation can be created. On the peer machine issuing the invitation, select a group for the identity to join. Then, right click the appropriate group and choose 'Create Invitation'. In the 'Create Invitation' dialog box, select the identity information transferred. In the same dialog box, click on the 'Browse' button next to the 'Invitation Location:' edit box, and choose a location for the new invitation. Type the invitation name in the 'Filename:' edit box and click 'OK'. Click 'OK' again in the 'Create an invitation' dialog box.

7. Join a Group
To join a group, transfer an invitation file via an email or file share to the machine of the peer for which the identity information was created. Select the invited identity.  Right click and select 'Join Group’. In the ‘Join Group’ dialog box, browse to the invitation created. Click 'OK' in the open dialog box when the invitation file is selected. Finally, click 'OK' again in the 'Join Group' dialog box. The identity has now joined the group.

8. Connect to a Group
To connect to a group, right click on the group to connect. Then, click the "Connect" submenu option.

9. Disconnect from a Group
To disconnect to a group, right click on the group to disconnect. Then, click the "Disconnect" submenu option.
